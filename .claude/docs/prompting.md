# Claude 프롬프트 가이드

이 저장소에서 Claude Code를 사용할 때는 다음 구조를 따르면 빠르게 정확한 답변을 받을 수 있습니다.

1. **목표 요약**: 한 문장으로 원하는 작업을 설명합니다.
2. **맥락 제공**: 영향을 받는 파일 경로와 중요 규칙(포인트는 100 단위, 최대 1,000,000)을 언급합니다.
3. **제약 조건**: 네트워크 사용 금지, ASCII 유지, ReentrantLock 전략 유지 여부 등을 명시합니다.
4. **검증 방법**: 반드시 실행해야 하는 테스트나 명령 (`./gradlew test`, 특정 테스트 클래스 등)을 나열합니다.
5. **산출물 형식**: 패치, 요약, 코드 리뷰 등 원하는 결과물을 명확히 합니다.

아래 템플릿을 바로 사용할 수 있습니다.

```
Goal: <간단한 설명>
Context:
- 관련 파일: <경로:라인>
- 도메인 규칙: 포인트는 100 단위, 최대 1,000,000, 동시성은 사용자별 잠금 유지
Constraints:
- 네트워크 접근 없음
- 새 의존성 추가 금지
Acceptance criteria:
- <기능 요구사항>
Validation:
- ./gradlew test --tests "..."
Deliverable: <patch | summary | review>
```

보다 긴 작업에는 저장소 루트의 `CLAUDE.md`에 있는 예시 템플릿을 참고하세요.
