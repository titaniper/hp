## STEP 19: 부하 테스트 스크립트 작성 및 진행

### 🎯 목표
부하 테스트 대상 선정 및 목적, 시나리오 등의 계획을 세우고 이를 문서로 작성

### 할일 목록

#### 1. 부하 테스트 계획 수립
- [ ] 테스트 대상 API 선정
  - 공통: 포인트 충전/사용 기능
  - 이커머스: 상품 주문 및 결제 (재고처리)
  - 대기열: 대기열 제어 로직 / 좌석 예약
- [ ] 목표 TPS 설정
- [ ] 예상 병목 지점 분석 (DeadLock, Slow Query)
- [ ] 테스트 시나리오 문서 작성

#### 2. 테스트 도구 선정 및 스크립트 작성
- [ ] 부하 테스트 도구 선택 (k6 / JMeter / nGrinder / Artillery)
- [ ] 테스트 스크립트 작성
- [ ] 테스트 환경 구성

#### 3. 테스트 실행
- [ ] Load Test 수행 (예상 부하 정상 처리 여부)
- [ ] Peak Test 수행 (폭발적 트래픽 내성 점검)
- [ ] 동시성 이슈 발생 여부 확인

#### 4. (NiceToHave) Docker 옵션 조정 테스트
- [ ] CPU/Memory 옵션 조정하며 성능 테스트
- [ ] 적절한 배포 스펙 도출

## STEP 20: 부하 테스트로 인한 문제 개선 및 보고서 작성

### 🎯 목표
테스트를 진행하며 획득한 다양한 성능 지표를 분석 및 시스템 내의 병목을 탐색 및 개선

### 할일 목록

#### 1. 성능 지표 분석
- [ ] TPS 분석
- [ ] 평균/중간/최대 응답시간 분석
- [ ] p95, p99 지표 분석
- [ ] 서버 리소스 메트릭 분석

#### 2. 병목 탐색 및 개선
- [ ] 병목 지점 식별
- [ ] 개선 방안 수립
- [ ] 개선 구현 및 적용
- [ ] 개선 후 벤치마크 테스트

#### 3. 장애 대응 문서 작성
- [ ] 장애 시나리오 정의
- [ ] 장애 대응 프로세스 문서화
  - 장애 탐지
  - 장애 분류 및 전파
  - 장애 복구 및 보고
  - 장애 상황 해소 통지
  - 장애 회고

#### 4. 장애 회고 문서 (포스트모템) 작성
```
장애리뷰 필수 항목

현상
- 타임라인
- 영향범위
- 고객영향도 (비즈니스 임팩트)

조치 내용
- 장애 원인
- 해소 타임라인
- 실제 단기 대응책
- 후속 대응 계획

분석
- 5-whys

대응 방안
액션 아이템
- short-term
- mid-term
- long-term
```

#### 5. 최종 보고서 작성
- [ ] 테스트 케이스 시나리오 설계부터 검증까지의 과정 문서화
- [ ] 휴먼 에러 발견 및 개선 과정 문서화
- [ ] 애플리케이션 에러 발생 통계 및 개선 과정 정리
- [ ] 문제 해결을 위한 개선 내용 정리
