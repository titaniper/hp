# Reports & Learning

이 디렉토리는 프로젝트 진행 과정에서의 학습 내용과 분석 리포트를 관리합니다.

## 디렉토리 구조

- **learning/**: 작업과 관련된 기술적 지식, 개념 학습 내용 정리
- **reports/**: 작업 수행 중 발생한 이슈 분석, 설계 결정, 성능 테스트 결과 등 공유가 필요한 리포트

---

## Report 작성 가이드

리포트 작성 시 아래 구조를 참고하여 작성합니다. 상황에 따라 유연하게 가감할 수 있습니다.

### 1. 개요 (Overview)

- 어떤 주제에 대한 리포트인지 간략하게 설명합니다.
- 예: "동시성 제어 방식에 따른 성능 비교 분석"

### 2. 배경 (Background)

- 이 리포트를 작성하게 된 계기나 상황을 기술합니다.
- 현재 시스템의 상태, 발생한 문제, 또는 새로운 요구사항 등을 언급합니다.

### 3. 목적 (Purpose)

- 이 분석이나 작업을 통해 달성하고자 하는 목표를 명확히 합니다.
- 예: "적절한 락 전략을 선정하여 데이터 정합성을 보장하면서 성능 저하를 최소화한다."

### 4. 문제 해결 과정 (Problem Solving)

- **분석**: 문제의 원인을 파악하거나, 여러 대안을 비교 분석한 내용을 기술합니다.
- **해결 방안**: 선택한 해결책과 그 이유를 설명합니다. 코드 스니펫이나 다이어그램을 활용하면 좋습니다.
- **구현**: 실제 적용한 방식에 대한 설명입니다.

### 5. 테스트 및 검증 (Test & Verification)

- 해결 방안이 유효한지 검증한 방법과 결과를 기술합니다.
- 테스트 케이스, 부하 테스트 결과(k6 등), 로그 등을 첨부합니다.

### 6. 한계점 (Limitations)

- 이번 해결책이 가지는 한계나, 아직 해결되지 않은 부분, 잠재적인 문제점 등을 솔직하게 기록합니다.

### 7. 결론 (Conclusion)

- 전체 내용을 요약하고, 최종적인 판단이나 배운 점을 정리합니다.

### 8. 향후 계획 (Next Steps)

- 한계점을 극복하기 위한 계획이나, 이어서 진행해야 할 작업들을 나열합니다.
